------------------------------------------------------------
2.1 USER AGREEMENT / TERMS OF SERVICE
------------------------------------------------------------

Назначение:
— описывает правила использования сервиса
— фиксирует границы ответственности
— защищает владельца продукта

Ключевые идеи:
— сервис предоставляется "как есть"
— аналитика носит рекомендательный характер
— результат зависит от внешних факторов
— владелец сервиса не несёт ответственности
  за коммерческие решения пользователя

Где используется:
— ссылка в боте
— кнопка в профиле
— согласие при первом использовании

------------------------------------------------------------
2.2 PRIVACY POLICY / ПОЛИТИКА КОНФИДЕНЦИАЛЬНОСТИ
------------------------------------------------------------

Назначение:
— объясняет, какие данные собираются
— зачем они используются
— как пользователь может прекратить использование

Пример данных:
— telegram_id
— username
— сообщения, отправленные боту
— статус Premium

Принципы:
— данные не продаются
— используются только для работы сервиса
— минимальный сбор информации

------------------------------------------------------------
2.3 DISCLAIMER / ОГРАНИЧЕНИЕ ОТВЕТСТВЕННОСТИ
------------------------------------------------------------

Назначение:
— юридически и этически снять ожидание "гарантий"

Формат:
— короткий дисклеймер внутри продукта
— расширенный текст в документах

Пример смысла:
"Аналитика — это помощь мышлению, а не истина.
Рынок изменчив. Данные могут быть неполными.
Окончательные решения принимает пользователь."

4. ЮРИДИЧЕСКАЯ ФОРМА (ИП / ООО / ОсОО)
------------------------------------------------------------

Принцип:
— юридическое оформление = функция дохода
— не наоборот

Когда есть:
— стабильный доход
— платные пользователи
— регулярные транзакции

Тогда:
— регистрация ИП / ОсОО
— подключение платежных систем
— уплата налогов

Патенты:
— для AI-бота не обязательны
— чаще регистрируют бренд (товарный знак)
